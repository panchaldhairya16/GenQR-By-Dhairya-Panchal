## ‚óą Feature Breakdown

### „ÄĒ 01 „Äē QR Code Generators

GenQR'By D ships four distinct, fully-typed generation modules ‚ÄĒ each tailored to a specific data schema and encoded with precision.

<br/>

#### ūüď∂ Wi-Fi QR Generator

> Encode complete wireless network credentials into a single scannable code ‚ÄĒ no manual password entry required.

- **SSID encoding** ‚ÄĒ supports all character sets including spaces, symbols, and Unicode
- **Hidden SSID toggle** ‚ÄĒ generates the `H:true` Wi-Fi QR schema for non-broadcasting networks
- **Security protocol selector** ‚ÄĒ WPA2, WPA3, WEP, and open network modes
- **Password field masking** ‚ÄĒ secure input with visibility toggle; never logged to local storage
- **Instant preview** ‚ÄĒ QR updates in real time as each field is populated
- **Copy-safe output** ‚ÄĒ exported codes are validated against IEEE Wi-Fi QR spec

<br/>

#### ūüĒó Custom Link / URL Generator

> Transform any URL ‚ÄĒ short, long, parameterized, or deep-linked ‚ÄĒ into a scannable QR code.

- **Raw URL input** with live format validation (HTTP, HTTPS, custom schemes)
- **UTM parameter builder** ‚ÄĒ append campaign tracking fields without breaking the QR structure
- **Deep link support** ‚ÄĒ handles `app://`, `mailto:`, `tel:`, and custom protocol prefixes
- **Character count indicator** ‚ÄĒ warn users when data density may affect scan reliability at small print sizes
- **Error correction level control** ‚ÄĒ choose between L, M, Q, H modes for optimal density vs. resilience tradeoff

<br/>

#### ūüďß Mail Setup Generator

> Pre-populate an outbound email with recipient, subject, and body ‚ÄĒ all encoded in a single tap.

- **To / CC / BCC field support** ‚ÄĒ multi-address entries separated by commas
- **Subject line encoder** ‚ÄĒ URL-safe encoding applied automatically
- **Body template composer** ‚ÄĒ multi-line rich-text entry with character limit guidance
- **`mailto:` schema compliance** ‚ÄĒ output conforms strictly to RFC 6068
- **Preview mode** ‚ÄĒ renders a decoded email card preview before final export

<br/>

#### ūüĎ§ Social Card / vCard Generator

> Generate a professional digital business card as a scannable QR code ‚ÄĒ importable into any contacts app.

- **vCard 3.0 / 4.0 schema** ‚ÄĒ full name, organization, title, phone, email, website
- **Social handle fields** ‚ÄĒ LinkedIn, GitHub, X (Twitter), Instagram
- **Profile photo URL encoder** ‚ÄĒ embed a hosted avatar reference inside the vCard payload
- **One-tap import compatibility** ‚ÄĒ tested across iOS Contacts, Android Contacts, and Outlook
- **Multi-language support** ‚ÄĒ Unicode-safe for names in Arabic, CJK, Cyrillic, and Devanagari scripts

## ‚óą The Tech Stack

> A curated, developer-first stack built for velocity, type safety, and zero-compromise performance.

<br/>

| Layer | Technology | Purpose |
|---|---|---|
| ‚öõÔłŹ **UI Framework** | React 19 | Component architecture, concurrent rendering |
| ūüĒ∑ **Type System** | TypeScript 5 (strict) | Full type coverage across all form states and data models |
| ūüé® **Styling** | Tailwind CSS v4 | Utility-first CSS with custom design token configuration |
| ‚ö° **Build Tool** | Vite 6 | Sub-100ms HMR, ES module-native dev server |
| ūüéěÔłŹ **Animations** | motion/react (Framer Motion) | Physics-based transitions, gesture handling, layout animations |
| ūüĒ≤ **QR Engine** | qrcode (npm) | Client-side QR matrix generation and SVG/PNG rendering |
| ūüíĺ **State** | React Hooks + localStorage API | Ephemeral UI state + persistent scan log storage |
| ūüöÄ **Deployment** | Vercel (SPA mode) | Edge-cached global distribution with instant rollbacks |

<br/>

### ‚öõÔłŹ React 19

React 19 introduces compiler-assisted rendering optimizations, making `useMemo` and `useCallback` largely redundant for standard component trees. GenQR'By D leverages:

- **`useOptimistic`** ‚ÄĒ for instantaneous QR preview updates before async operations settle
- **`useTransition`** ‚ÄĒ non-blocking input handling ensuring the UI never freezes during heavy QR matrix computation
- **`useActionState`** ‚ÄĒ for declarative form error handling across all four generator modules
- **Concurrent rendering** ‚ÄĒ prioritized paint order ensures the QR canvas updates before secondary UI elements

<br/>

### ūüĒ∑ TypeScript 5 ‚ÄĒ Strict Mode

Every data boundary in GenQR'By D is typed. No `any`. No implicit `unknown`. Full strictness flags enabled in `tsconfig.json`:

```json
{
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true,
    "noImplicitReturns": true,
    "noFallthroughCasesInSwitch": true
  }
}
```

Key typed interfaces include `WiFiConfig`, `MailConfig`, `VCardPayload`, `ScanLogEntry`, and the `QRRenderOptions` discriminated union that powers the generation engine.

<br/>

### ūüé® Tailwind CSS v4

Tailwind v4 replaces `tailwind.config.js` with a CSS-first configuration model. Custom tokens are declared directly in the stylesheet:

```css
@import "tailwindcss";

@theme {
  --color-void:    #0F172A;
  --color-cyan:    #22D3EE;
  --color-violet:  #A855F7;

  --font-display:  "Space Grotesk", sans-serif;
  --font-body:     "Inter", sans-serif;
  --font-mono:     "JetBrains Mono", monospace;
}
```

This approach eliminates JavaScript configuration overhead and enables native CSS cascade layers for deterministic specificity control.

<br/>

### ūüéěÔłŹ motion/react (Framer Motion)

Animations are not cosmetic in GenQR'By D ‚ÄĒ they are functional UX signals:

- **Glowing border pulse** ‚ÄĒ activates on QR canvas render completion, signaling readiness
- **Sliding toast notifications** ‚ÄĒ success, warning, and error states animate in from the bottom-right with spring physics
- **Input type switcher** ‚ÄĒ layout animations with `AnimatePresence` ensure smooth generator panel transitions with no layout shift
- **Micro-movements** ‚ÄĒ subtle scale and opacity shifts on interactive elements reinforce affordance without distraction

<br/>

### ‚ö° Vite 6 Dev Server

```
Dev server cold start:    < 300ms
Hot Module Replacement:   < 50ms per module
Production build time:    < 8s (typical project scale)
Chunk splitting:          Automatic via Rollup
```

Vite resolves ES modules natively in development ‚ÄĒ no bundling step, no waiting. The `qrcode` package is pre-bundled via `optimizeDeps` to eliminate re-compilation on every import.

<br/>

### ūüĒ≤ qrcode Engine

The `qrcode` package handles all matrix generation logic client-side:

- **Error correction levels**: L (7%), M (15%), Q (25%), H (30%) ‚ÄĒ configurable per generator type
- **Version auto-detection**: selects the minimum QR version that fits the encoded payload
- **SVG renderer**: outputs clean, minimal SVG with configurable module sizing and quiet zone
- **Canvas renderer**: used for PNG export via `toDataURL()` at configurable DPI

## ‚óą Vercel Deployment Blueprint

GenQR'By D is architected as a Single Page Application. All routing is handled client-side. Vercel must be configured to serve `index.html` for all routes.

<br/>

### Method A ‚ÄĒ GitHub Repository Sync (Recommended)

**1. Import the repository**

Navigate to [vercel.com/new](https://vercel.com/new) ‚Üí **Import Git Repository** ‚Üí Select `genqr-by-d`

**2. Configure build settings**

Vercel auto-detects Vite. Confirm the following:

| Setting | Value |
|---|---|
| **Framework Preset** | Vite |
| **Build Command** | `npm run build` |
| **Output Directory** | `dist` |
| **Install Command** | `npm install` |
| **Node.js Version** | 20.x |

**3. Add the SPA rewrite rule**

Create a `vercel.json` at the project root:

```json
{
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```

**4. Deploy**

Click **Deploy**. Vercel will build and publish to a `.vercel.app` domain with global edge distribution. Every subsequent push to `main` triggers an automatic redeployment.

<br/>

### Method B ‚ÄĒ Vercel CLI (Direct Push)

```bash
# Install the Vercel CLI globally
npm install -g vercel

# Authenticate with your Vercel account
vercel login

# Deploy from the project root (follows vercel.json config)
vercel --prod
```
